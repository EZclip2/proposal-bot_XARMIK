# Идеальный бот-предложка

Telegram-бот приёма предложений с **одной редактируемой карточкой в модерации** и ответами через **мини-приложение**.

```
Подписчик (личка)              Модерационная группа
─────────────────              ─────────────────────
пишет предложение  ──────►     🟡 Новое №142   [Ответить] [Отклонить]
                                     │  открывается мини-приложение
                                     ▼  (та же карточка редактируется)
                               🟠 Обрабатывается · @moderator
                                     │  модератор пишет ответ
                                     ▼  (та же карточка редактируется)
получает ответ  ◄──────────    🟢 Обработано + текст ответа
```

---

## Структура файлов

```
proposal-bot/
├── bot.js            ← основной файл бота
├── package.json
├── .gitignore        ← содержимое файла gitignore.txt (переименовать!)
├── .env.example      ← содержимое файла env.example.txt (переименовать!)
└── public/
    └── webapp.html   ← мини-приложение (положить в папку public/)
```

> На GitHub: создайте файл с именем `.gitignore` (содержимое из `gitignore.txt`)
> и `.env.example` (содержимое из `env.example.txt`). Папку `public/` создайте вручную.

## Шаг 1. Подготовка в Telegram

### 1.1 Создать бота
1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Укажите имя и username. BotFather выдаст **BOT_TOKEN** — сохраните.

### 1.2 Зарегистрировать мини-приложение
1. Напишите BotFather → `/newapp`.
2. Выберите только что созданного бота.
3. Введите любое **короткое имя** (например: `reply`) — это `APP_NAME`.
4. Когда попросит **Web App URL** — введите временную заглушку: `https://example.com`
   (замените после получения адреса Render).
5. Загрузите любое изображение 640×360 (обязательное поле).

### 1.3 Создать модерационную группу
1. Создайте супергруппу, добавьте бота и сделайте его **администратором**
   (нужно право на удаление сообщений).
2. Узнайте ID группы через [@getidsbot](https://t.me/getidsbot) — вида `-1002033645033`.

## Шаг 2. Загрузка кода на GitHub

1. Откройте [github.com](https://github.com) → **New repository** → имя `proposal-bot` → **Create**.
2. Нажмите **uploading an existing file** и загрузите:
   - `bot.js`
   - `package.json`
   - Создайте файл `.gitignore` вручную (New file) — скопируйте содержимое из `gitignore.txt`
   - Создайте файл `.env.example` вручную — скопируйте содержимое из `env.example.txt`
   - Создайте `public/webapp.html` вручную — введите имя `public/webapp.html` и вставьте содержимое
3. Нажмите **Commit changes**.

---

## Шаг 3. Деплой на Render

1. Откройте [dashboard.render.com](https://dashboard.render.com) → **Sign in with GitHub**.
2. **New +** → **Web Service** → выберите `proposal-bot` → **Connect**.
3. Настройки:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
4. **Advanced** → **Environment Variables** → добавьте **4 переменные**:

   | Key | Value |
   |-----|-------|
   | `BOT_TOKEN` | токен от BotFather |
   | `MODERATION_CHAT_ID` | ID группы, например `-1002033645033` |
   | `BOT_USERNAME` | username бота без `@` |
   | `APP_NAME` | короткое имя мини-приложения, например `reply` |

5. **Create Web Service** → дождитесь строки в логах:
   ```
   Вебхук установлен: https://proposal-bot.onrender.com/bot...
   ```
6. Скопируйте адрес сервиса (вида `https://proposal-bot.onrender.com`).

---

## Шаг 4. Обновить URL мини-приложения в BotFather

1. BotFather → `/myapps` → выберите бота → приложение → **Edit Web App URL**.
2. Введите: `https://proposal-bot.onrender.com/webapp.html`

---

## Шаг 5. Anti-Sleep (cron-job.org)

1. Откройте [cron-job.org](https://cron-job.org) → **Create Cronjob**.
2. **Address**: `https://proposal-bot.onrender.com`
3. **Schedule**: Every **12 minutes** → **Create**.

---

## Проверка

1. Напишите боту `/start` — появится меню.
2. Нажмите «✍️ Предложка» → напишите что-нибудь.
3. В модерационной группе появится карточка с кнопками **Ответить** и **Отклонить**.
4. Нажмите **Ответить** — откроется мини-приложение.

---

## Важные детали

- **Кнопка «Ответить» — ссылка**, а не web_app-кнопка: обычные web_app-кнопки не работают в группах.
- **`data.json`** хранится на диске контейнера и не стирается при пробуждении после сна. Стирается только при новом деплое.
- Ответ модератора проверяется по подписи `initData` — только реальные участники группы могут отвечать через мини-приложение.
- Медиа-карточки: подпись Telegram ограничена 1024 символами — очень длинный ответ в карточке обрезается, но подписчику приходит полностью.
